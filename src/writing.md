# ドキュメントの書き方 (Writing Guide)

Atlasでのドキュメント作成は、基本的に **`src/` ディレクトリ内の Markdown ファイル（`.md`）だけを編集** していくスタイル（Docs as Code）です。
設定ファイル（TOML）やCSS、JavaScriptを普段触る必要はありません！

## ページの追加手順

1. `src/` ディレクトリ内に新しい Markdown ファイル（例: `api_reference.md`）を作成し、中身を書きます。
2. `src/SUMMARY.md` を開き、作成したファイルへのリンクを箇条書きで追加します。

**SUMMARY.md の例:**
```markdown
- [APIリファレンス](./api_reference.md)
```

## ローカルでのプレビュー（執筆用コマンド）

執筆中のドキュメントをブラウザでリアルタイムに確認するために、便利な `make` コマンドが用意されています。ターミナルで以下を実行してください。

```bash
make
```
（※または `make serve`）

このコマンドを叩くと、以下の処理が自動で行われます。
1. 共通テーマ（`atlas-common`）に更新があれば最新版を取得する
2. ローカルサーバーが起動し、自動的にブラウザが開く（`http://localhost:3000`）

一度開いたら、あとは **Markdownファイルを保存するたびにブラウザの画面が自動更新** されるため、サクサク執筆が進みます！

### その他の便利なコマンド

- `make build` : 本番環境と同じ静的HTMLファイルを `book/` フォルダに手動で生成します（デプロイ前の最終確認用）。
- `make clean` : 生成された `book/` フォルダを削除してお掃除します。

## Markdownの書き方

通常のMarkdown記法（見出し、リスト、リンク、コードブロックなど）がすべて使用可能です。さらにmdbook固有の機能も利用できます。詳細は [mdbookの公式ドキュメント](https://rust-lang.github.io/mdBook/) をご参照ください。

### 警告ボックス（独自のハイライト）

Atlas独自のスタイルとして、`>`（引用符）を使ったブロッククオートは、以下のような「ハイライトボックス」として美しく表示されます。
さらに、**最初の文字**に応じて自動的に色が切り替わるようになっています！

#### 1. デフォルト（参考・メモなど） - 青系
> **【参考】**
> 先頭に `>` をつけるだけで、テーマ（ライト/ダーク）に合わせた青色のデザインが適用されます。
> 補足事項などを書くのに最適です。

#### 2. 注意（Caution） - 黄色系
> **【注意】**
> 先頭が `【注意】` や `[!CAUTION]` で始まると、自動的に黄色系のボックスになります。
> 少し気を付けてほしいポイントなどで使用してください。

#### 3. 警告（Warning） - 赤系
> **【警告】**
> 先頭が `【警告】` や `[!WARNING]` で始まると、赤色系のボックスになります。
> 重大なトラブルにつながる事項など、絶対に読んでほしい箇所に使用してください。

### その他の標準拡張スタイル

Atlas独自の標準テーマ（`atlas-common`）には、マニュアル作成に役立ついくつかの便利な装飾が標準搭載されています。

#### 1. 折りたたみメニュー（Details / Summary）

長いエラーログや、補足事項を隠しておきたい場合に使います。HTMLタグをそのまま記述します。

<details>
<summary>クリックして詳細を開く</summary>

このように、クリックすると中身が展開されるアコーディオンメニューになります。
長くなりがちな「トラブルシューティング」などで非常に役立ちます。

</details>

#### 2. キーボードショートカットの装飾

マニュアルでキー操作を説明する際は、`<kbd>` タグを使用すると物理キーボードのような立体的な装飾が自動で適用されます。

例：コピーするには <kbd>Ctrl</kbd> + <kbd>C</kbd> を押してから、<kbd>Ctrl</kbd> + <kbd>V</kbd> を押します。

#### 3. 画像（スクリーンショット）の美化

マークダウンで画像を挿入すると、自動的に「角丸」と「うっすらとしたドロップシャドウ」が適用され、画面から少し浮き出たようなリッチな見た目になります。設定は不要で、普通に画像を配置するだけで適用されます。

#### 4. 編集者向けの非表示メッセージ（インクルード時の工夫）

ファイルを分割して `\{{#include api_user.md}}` のように読み込むと、画面右上の「鉛筆アイコン」を押して編集画面を開いた際に「どこを編集すればいいか分からない」状態になりがちです。

この問題を解決するため、**「閲覧者には見えないが、編集画面（Markdownソース）を開いた人にだけ見える」** 特別なクラス `.editor-only` を用意しています。

```markdown
<div class="editor-only">

💡 **この部分は共通パーツを読み込んでいます！**
編集する場合は、上の鉛筆アイコンではなく [📝 こちらのリンク（api_user.md）](https://github.com/totototo0526/atlas/edit/main/src/api_user.md) をクリックして直接ファイルを編集してください。

</div>

\{{#include api_user.md}}
```

上記のように記述しておくと、完成したマニュアル上では完全に非表示になりますが、編集画面を開いたエンジニアには巨大な案内書きとリンクが目に入るため、迷わず正しいファイルを編集することができます！

#### 5. 印刷時の改ページ（PDF出力用）

ブラウザの印刷機能（PDFとして保存）を使ってマニュアルを出力する際、意図した場所でページを変えたい場合は、Markdown内の好きな場所に以下のタグを挿入します。

```html
<div class="page-break"></div>
```

このタグを置いた直後で、印刷時に必ず改ページが行われます（画面上の見た目には何も影響を与えません）。章の区切りなどで強制的にページを変えたい場合に便利です。
