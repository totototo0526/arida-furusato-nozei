# Agent Context (Current Status & Core Philosophy)

本ファイルは、Atlas Systemプロジェクトにおいて、開発者とAIアシスタント（Agent）がプロジェクトの現状や設計思想を共有・同期するためのコンテキストファイルです。

## Core Philosophy (設計思想)

1. **自動化と省力化の徹底**: デプロイメントから運用まで、極力手作業を排除し、GitHub Actions等を活用したCI/CDパイプラインを構築する。
2. **無停止・安全なリリース**: ブルーグリーン・デプロイメント（シンボリックリンク切替）を採用し、ユーザー影響ゼロでアップデートを行う。
3. **ドキュメント・ドリブン**: mdbookを活用し、仕様やトラブルシューティング手順をコードと共に管理・更新する。
4. **日本語ベースのコミュニケーション**: 開発・運用に関する主要ドキュメント（Plan, Task, Walkthrough等）は日本語で作成し、`docs/` 配下にトピックごとに整理する。

## Current Status (現在のシステム状態)

- **自動デプロイコンベアの稼働**: VPSへの無停止デプロイメントパイプラインが安定稼働中。
- **マニュアル基盤（Atlas）の完成**: `mdbook` を用いたマニュアル構成、共通金型（`atlas-common`）へのCSS/JS分離、GitHub Webエディタ連携、完全部分一致の日本語全文検索エンジン導入が完了しました。

## Future Architecture (次期アーキテクチャ構想: Workspaceモノレポ化と波及ビルド)

将来的に、単なる静的マニュアルを超えた「双方向な社内ポータル（UIからの入力、裏側での自動コミット連携など）」へと進化させるため、以下のアーキテクチャを構想・記録しています。

1. **`atlas-common` の Cargo Workspace 化 (モノレポ)**
   - `atlas-common` リポジトリを拡張し、フロントエンドテーマ（JS/CSS）だけでなく、裏側で動く **APIサーバー** や **独自Preprocessor（Rust）** も同一リポジトリ内に内包させます。
   - これにより、「APIの仕様変更」と「フロント画面の改修」が常に1コミットで完全に同期し、エコシステム全体のバージョン管理が極めて強固になります。
   - *※非エンジニアの執筆者は画面（Web UI）から操作し、APIが裏でコミットを代行する運用を想定しているため、執筆者のPCにRust環境を強要することはありません。*

2. **`repository_dispatch` を用いた「波及ビルド機構（全一斉アップデート）」**
   - APIサーバーや共通JSがアップデートされた際、末端の各マニュアルリポジトリ（`atlas`等）が古いサブモジュールを参照したままになり、「古いJSから新しいAPIを叩いてエラーになる」というバージョン乖離のサイレント障害を未然に防ぎます。
   - `atlas-common` のビルド・デプロイ完了時に、すべての子リポジトリへWebHook（`repository_dispatch`）を送信し、**すべての子リポジトリが一斉に最新のサブモジュールを取り込んで自動再ビルドされる仕組み** を構築します。
